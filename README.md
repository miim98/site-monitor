# site-monitor

지정한 웹페이지들의 **글 목록을 GitHub Actions가 주기적으로 가져와**,
직전까지 본 적 없는 **새 글(상세 링크)만** 골라 기록해두는 도구입니다. 알림은 보내지 않습니다.
변경 내역은 **웹 대시보드** 또는 **클로드 코드에서 레포를 열어** 확인합니다.
기록은 **14일(2주)이 지나면 자동으로 삭제**되어 항상 최근 2주치만 유지됩니다.

---

## 동작 방식

매 실행마다:

1. `config.json`의 사이트에서 글 목록의 `{제목, 상세링크}`를 가져온다
2. 사이트별로 이미 본 링크(`seen.json`)와 비교한다
3. 새로 생긴 항목만 **감지 시각**과 함께 `changes.json`에 한 건씩 기록한다
4. 14일(2주) 지난 항목은 삭제한다
5. 사람이 읽기 좋은 `CHANGES.md`와 빈 화면용 `latest.json`을 다시 만든다

> 판별 기준이 되는 `seen.json`은 기간제로 지우지 않습니다(이미 본 링크 누적).
> 기간제로 삭제되는 건 '감지된 새 항목 목록'뿐입니다.
>
> 글 항목이 `<a>` 링크가 아니라 JS 버튼인 사이트(예: 일부 포트폴리오)는 상세 링크를
> 뽑을 수 없어, 대시보드에서 '감시 사이트' 카드로만 표시됩니다.

## 감시할 사이트 추가하기

`config.json`의 `sites`에 항목을 추가하면 됩니다. 코드는 건드릴 필요 없습니다.

```json
{
  "retention_days": 7,
  "sites": [
    {
      "name": "회사-공지",
      "url": "https://example.com/notice",
      "selector": null
    }
  ]
}
```

- **name**: 구분용 이름(파일명/표시에 쓰임). 한글 가능.
- **url**: 감시할 페이지 주소.
- **selector**: 페이지에서 **볼 영역**을 좁히는 CSS 선택자. 페이지 전체를 보려면 `null`.
  - 예: 공지 목록만 보고 싶으면 `".notice-list"`, 본문만 보려면 `"#content"` 등.
  - 영역을 좁힐수록 광고·날짜 같은 잡음으로 인한 오탐이 줄어듭니다.
  - 어떤 선택자를 넣어야 할지 모르겠으면, 그 페이지 HTML을 클로드 코드에 주고
    "이 목록만 잡는 CSS 선택자 알려줘"라고 하면 됩니다.

## 처음 설정 (한 번만)

1. 이 폴더를 GitHub 레포지토리로 올린다(push).
2. 레포의 **Settings → Actions → General → Workflow permissions**에서
   **Read and write permissions**를 켠다(봇이 변경분을 커밋할 수 있어야 함).
3. **Actions** 탭에서 워크플로를 활성화한다.
4. 첫 실행은 **Actions 탭 → site-monitor → Run workflow**로 수동 실행하면 됩니다.
   - 첫 실행은 비교 대상이 없으므로 "기준 스냅샷 저장"만 하고 변경 기록은 없습니다.
   - 두 번째 실행부터 변경을 잡기 시작합니다.

실행 주기는 `.github/workflows/monitor.yml`의 `cron` 값으로 조절합니다(기본: 매시간).

## 변경 내역 확인 (웹 대시보드)

최근 변경 내역을 카드로 보여주는 정적 대시보드가 GitHub Pages로 배포됩니다.

- 주소: <https://miim98.github.io/site-monitor/>
- `docs/index.html` 한 파일로 동작하는 정적 페이지이며, `docs/changes.json`을 읽어 최신순 카드로 표시합니다.
- 카드를 누르면 해당 사이트가 새 탭에서 열립니다. 사이트별 필터도 됩니다.
- monitor 워크플로가 돌 때마다 `changes.json`을 `docs/`로 복사해 커밋하므로 페이지가 자동 갱신됩니다.

> 참고: 무료 플랜에서는 Pages를 쓰려면 레포가 **public**이어야 합니다(이 레포는 그래서 공개 상태).

## 변경 내역 확인 (클로드 코드에서)

레포를 클로드 코드로 열고 이렇게 물어보면 됩니다.

- "CHANGES.md 열어서 최근에 뭐 바뀌었는지 정리해줘"
- "지난 며칠간 어떤 사이트에 새 글이 올라왔어?"

`CHANGES.md`만 봐도 사람이 바로 읽을 수 있고,
`changes.json`은 클로드 코드가 더 정밀하게 분석할 때 쓰는 원본 데이터입니다.

## 알아둘 점

- **자바스크립트로 그려지는 사이트**: 이 도구는 정적 HTML 기준입니다. 어떤 사이트의
  스냅샷이 비어 있거나 이상하면 그 사이트가 JS 렌더링 방식일 가능성이 큽니다.
  그 경우 클로드 코드에 "이 사이트는 Playwright로 가져오게 바꿔줘"라고 하면 됩니다.
- **스케줄 비활성화**: 깃허브는 레포에 60일간 활동이 없으면 예약 워크플로를
  자동 중단합니다. 가끔 커밋이 쌓이니 보통은 문제되지 않습니다.
- **무료 한도**: 이 정도 작업은 GitHub Actions 무료 사용량 안에서 충분히 돕니다.
